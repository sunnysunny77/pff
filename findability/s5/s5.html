How to use Traffic Analysis Tools

Daniel Costello

What are Traffic Analysis Tools

Web analytics tools provide data on which to analyze involving website traffic for domains. This data then can be monitored so that marketing based adjustments and other changes can be optimised on the domain to improve the data reports and achieve goals.

For example, a common type of data reported on is visitors, if the data shows that no page visits come from a specific geographic area, then this area can be focused upon.

The two main ways to collect data

There are two main ways in which a domain's data can be collected.

1. This being through tagging pages with pieces of code, in order to send HTTP data to an external analytics platform.

2. Or secondly, having an application directly analyze the servers log files to propdce a report.

Pros and Cons Collecting Data

As the log file method is run independently from the browser it leads to a faster result, with no need for additional http communications that might otherwise fail. Log files also collect all data on top of human made searches, not excluding results from things such as web crawlers.

Page tags are however dependent on browsers. This can lead to delays in loading the page initially. While the platforms themselves also are delayed in producing reports, which is typically a few hours. Furthermore, the browser may somehow be unable to execute the page tag communication request, resulting in the data set being incomplete . On the other hand, browser information can be collected and added to the data, such as resolutions, or other page interactions which log files do not have access to.

Metrics

Metics provide information about a process or activity from numbers. There are many available metrics to focus on however in traffic analysis common metric categories are.

•Audience
The number of views received from specific demographics.

•Acquisition
Where visitors arecoming from, possibly from a direct search, though an advertisement, or external referral links.

•Sessions
How long a visitor is spending on the domain.


•Pageviews
The specific amount of views single pages on a domain receive.

•Bounce Rate
When visitors enter and exit the domain from the same page.

While optimising a domain, target goals can be set on metrics to act as an indication of performance. This feedback system can be used to make relevant changes in order to meet these goals.

How to integrate Google Search Console

Google search Console helps monitor, maintain, and troubleshoot SEO findability for your domain.


(About Search Console - Search Console Help, 2021)

'Confirm that Google can find and crawl the site.'

'Fix indexing problems and request re-indexing'

'Traffic data: how often the site is searched, search queries showing the site, how often searchers click through the queries, and more.'

'Receive alerts when Google encounters indexing, spam, or other issues'

'Show you which sites link to the website.'

'Troubleshoot issues for AMP, mobile usability, and other Search features.'


The first step in facilitating google search console on a domain is to sign up for the free platform at:


https://search.google.com/search-console

Next ownership of the domain needs to be verified by starting a new property inside the console, and then adding the page tag or other verification method to the site, uploading it to your web server. Now is also a good time to upload robots.txt and sitemap.xml files as these will also be checked by the console.


More information on robots files
More information on sitemap files


The next step involves adding a domain url to the google search index, and consequently start collecting SEO data. The URL inspection tool is used to test the url and provide feedback on its SEO status, possible issues, and whether or not it can be indexed. If all of the HTML is up to Google's SEO standards, you can then request that the url be indexed.


More information


After approximately a 48 hour wait the URL should be findable on google. And metric data for analysing can start to be seen in the console.

How to integrate GoAccess web log Analyzer

Go access provides a real time http statistics page generated from the web servers log file. In order to set up GoAccess on a Ubuntu server using Nginx as a web server, the following steps can be taken.

The directive:
“access_log /var/log/nginx/access.log;”

Should be added to the domain's server block so that the web server will record the logs for the domain in that file. I will not be covering how to set up server blocks. However,here are some links to relevant tutorials.

Learn how to set up Server blocks.
Learn how to set up HTTPS.


It is possible to then proxy a report page from the same domain server block. However it is probably better to keep the live report url separate from the main domain. So in a second server block for the report, the following location blocks should be included.

“location / { try_files $uri/report.html =404; }”

This will enable the report.html file to be served on the root of the domain or else show the 404 page.

“location /ws { proxy_pass http://localhost:7890; }”

This will proxy the live websocket to the browser to local service from the next step.


To actually run the live service on the local host port, the following commands are used in Ubuntu's command line.
“apt-get install goaccess”
To install the application.

“goaccess /var/log/nginx/access.log -o /home/admin/web/go.sunnyhome.site/public_html/report.html --log-format=COMBINED --real-time-html --ws-url=wss://go.sunnyhome.site/ws”

This example links the log output to the root directory that the web server is serving, and listens for the secure “wss” websocket url on location “/ws”. This command is best added as a system service so that it is always running, however I will not be covering how to do that.
